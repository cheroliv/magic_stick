== CI/CD — Limitations connues

=== A/B Partition Test (isoTestAB)

**Probleme** : Les runners GitHub Actions (`ubuntu-latest`) ne sont **pas privileged**.  
Le test `isoTestAB` requiert des loop devices (`losetup`, `parted`, `mkfs`, ...) → `CAP_SYS_ADMIN`.  

**Decision** (Session 020) :
- Step `isoTestAB` skippee dans `.github/workflows/build.yml` (`if: false`)
- Mock test en remplacement : `bash -n` sur `test-ab-partition.sh` et `update-system.sh` + verification presence task Gradle
- Voir `.agents/AB_PARTITION_REFERENCE.adoc` -- CI/CD Limitations pour les details runner self-hosted

=== Persistence Survival Mock Test (Session 021)

**Probleme** : Comme `isoTestAB`, le test complet de survie persistence requiert un loop device.

**Solution** : Test static (mock) valide en CI/CD non-privileged :
1. Verifier que `PERSISTENCE_LABEL="persistence"`
2. Verifier qu'aucun `mkfs`, `dd`, `parted mkpart` ne cible `persistence` en dehors de `cmd_setup_ab`
3. Verifier que `mount` de partition 3 est toujours `-o ro` en dehors du setup
4. Verifier que `cmd_install` et `cmd_update` ne referencent jamais la partition 3 (hors messages utilisateur)
5. Verifier la presence des messages de securite utilisateur

**Script** : `scripts/test-persistence-mock.sh`

**Execution CI/CD** :
[source,bash]
----
bash scripts/test-persistence-mock.sh scripts/update-system.sh
----

**Execution en local** :
[source,bash]
----
./gradlew isoTestAB
----
