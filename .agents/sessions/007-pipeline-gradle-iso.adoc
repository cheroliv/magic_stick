== Decisions cles

=== Zero sudo, zero install hote

* Tout s'execute dans un seul container Docker
* build.sh (hote) lance un seul `docker run` qui execute build-inner.sh
* verify.sh et test-boot.sh autodetectent `/.dockerenv` — si hors container, re-executent dans Docker via `exec docker run`
* Pas de Docker-in-Docker

=== Orchestration Gradle

* 10 tasks dans le groupe `iso`
* Tasks utilisees en local : `./gradlew isoPipeline` (build + verify + test)
* Tasks utilisees en CI : `./gradlew dockerBuild isoBuild isoVerify isoTestBoot`
* isoFlash : `./gradlew isoFlash -Pdevice=/dev/sdX`
* Tasks Exec Gradle natives (pas de doLast + project.exec)
* bakery + a11y preserves (site docs)

=== CI/CD GitHub Actions

* Checkout + JDK 17 + Gradle actions/setup-gradle
* Pipeline : dockerBuild → isoBuild → isoVerify → isoTestBoot → upload artifact → release (si tag v*)
