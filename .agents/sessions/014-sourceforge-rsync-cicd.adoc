== Lecons apprises

* Cloudflare R2, Workers, D1, Images necessitent tous une CB — le "free tier" cache cette contrainte
* SourceForge : gratuit, pas de CB, SFTP+rsync, pas de limite de taille stricte, mais pas d'API S3
* rsync est un outil solide pour le transfert CI/CD — incremente, reprise sur erreur, checksum
* Une ISO squashfs est deja compresssee en interne : xz par-dessus ajoute peu de gain mais oblige un decompress avant dd
* SourceForge = ISO brute (pret a flasher) + xz (compact pour le telechargement)
* Garder le .xz pour GitHub Releases (limite ~2 Go par asset), mais fournir l'ISO brute sur SourceForge
* L'arborescence du README doit reflechir la structure reelle du repo (build-inner.sh, test-boot.sh, etc.)
