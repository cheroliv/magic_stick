== Lecons apprises

* Les symlinks absolus dans un squashfs/extraction temporaire sont **cassés** car ils pointent vers la racine du système hôte, pas le répertoire d'extraction
* `[[ -e symlink_cassé ]]` retourne `false` en bash — il faut soit utiliser `[[ -L ]]` pour vérifier l'existence du lien, soit résoudre la cible relativement
* Le pattern d'installation locale `~/apps/` pour VS Code suit le même principe que l'installation `~/apps/gh` — binaires modifiables sans sudo
