== Bugs Identifiés

=== 1. build.yml ne se déclenche PAS sur tag v0.1.4

**Symptôme** : `release.yml` pousse le tag `v0.1.4` avec `GITHUB_TOKEN`, mais `build.yml` ne se trigger pas.

**Cause racine** : GitHub n'autorise pas les workflows à se déclencher mutuellement via `GITHUB_TOKEN` pour éviter les boucles infinies.

**Solution appliquée** : Utiliser `GH_PAT` (Personal Access Token) dans `release.yml` au lieu de `GITHUB_TOKEN`.

**État** : Fix appliqué localement (`release.yml`). **Non testé en CI** (nécessite le secret `GH_PAT` configuré dans Settings).

=== 2. ISOLINUX fichiers .c32 absents de l'ISO

**Symptôme** : `verify.sh` déclare 4 FAIL : `ldlinux.c32`, `libcom32.c32`, `libutil.c32`, `vesamenu.c32`.

**Cause racine** : Le hook `005-fix-syslinux-paths.chroot` crée des **liens symboliques**, mais `lb binary_syslinux` + `xorriso` ne copie pas les symlinks dans l'ISO. Les fichiers physiques ne sont donc pas présents.

**Workaround appliqué** : `verify.sh` passe de `FAIL` à `WARNING` sur les `.c32` (l'ISO boot toujours via GRUB2/UEFI, ISOLINUX n'est pas le bootloader principal).

**Vrai fix à implémenter** : Remplacer les `ln -s` par des `cp` physiques dans `005-fix-syslinux-paths.chroot`.

=== 3. pnpm absent de l'ISO (test-software.sh échoue)

**Symptôme** : `[FAIL] pnpm NOT found` dans les logs du build ISO.

**Cause racine** : Le hook `060-install-chroot-software.chroot` appelait l'API GitHub non-authentifiée (`/releases/latest`) → rate limit **60 req/h** → retourne tag fallback incorrect → curl 404 → pnpm non installé.

**Solution appliquée** : Hardcoding de la version pnpm (`v10.33.2`) avec download direct, bypass complet de l'API GitHub.

**État** : Fix appliqué localement. **Non testé en CI**.

== Leçons apprises

* GitHub ne permet pas aux workflows de se déclencher entre eux via `GITHUB_TOKEN` → nécessite PAT
* Les API GitHub non-authentifiées ont un rate limit très bas (60 req/h) dans les CI → hardcoder les versions ou utiliser une action de cache
* Les symlinks dans les hooks live-build ne sont pas copiés dans l'ISO par `lb binary_syslinux`/`xorriso`
