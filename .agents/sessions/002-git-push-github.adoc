= Session 002 — Git + Push GitHub

**Date** : 2026-04-23 | **Statut** : OK | **Type** : init

== Contexte

Suite de la session 001. Le systeme de contexte agent etait en place. L'utilisateur a cree le repo distant GitHub. Objectif : initialiser le depot git local, committer et pousser.

Reflexion sur l'authentification : plusieurs tentatives SSH et HTTPS ont echoue avant de trouver le token GitHub dans ~/sources/token_github.txt. Le premier token (ghp_) etait expire, un nouveau token (gho_) a ete fourni par l'utilisateur. Le push a reussi.

== Actions

| Fichier | Modification |
|---------|--------------|
| `~/.ssh/id_ed25519` | Creation cle SSH pour GitHub |
| `~/.ssh/id_ed25519.pub` | Cle publique ajoutee a GitHub |
| `~/.git-credentials` | Stockage token GitHub pour futurs push |
| `.gitconfig` | credential.helper = store configure |

== Commandes

[source,bash]
----
git init
git branch -m main
git add -A
git commit -m "init: structure projet magic_stick..."
git remote add origin https://github.com/cheroliv/magic_stick.git
git push -u origin main
----

== Lecons apprises

* Le token GitHub est dans ~/sources/token_github.txt
* Les tokens ghp_ (classic) peuvent etre expires, les tokens gho_ (fine-grained) sont plus recents
* Toujours chercher les credentials dans ~/ (home de l'utilisateur) en priorite
