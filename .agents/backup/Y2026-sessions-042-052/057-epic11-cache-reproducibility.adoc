== Taches realisees

=== Cache-BUILD (3 strates)

**Strate 1 — Cache GitHub Actions `build/cache/`** :
- `.github/workflows/build.yml` : cache key etendu avec `scripts/lib/versions.sh` (changement de version → invalidation cache)
- Key : `lb-cache-v2-${{ runner.os }}-${{ hashFiles('package-lists/**', 'VERSION', 'versions.sh') }}`

**Strate 2 — Cache Docker builder layers** :
- `.github/workflows/build.yml` : `docker/build-push-action` avec `cache-from: type=gha,scope=builder` + `cache-to: type=gha,scope=builder,mode=max`
- `build.gradle.kts` : `dockerBuild` devient no-op si l'image existe (`docker image inspect`)

**Strate 3 — Cache downloads externes** :
- `scripts/build-inner.sh` : stage/collect `build/cache/downloads/` vers/depuis le chroot avant/apres `lb build`
- `060-install-chroot-software.chroot` : fonction `dl_cached()` utilisee par tous les telechargements :
  - UV, ripgrep, fd, lazygit, just, xh, opencode, JetBrains Toolbox, VS Code, gh CLI
  - Si fichier present dans `/tmp/dl-cache/`, copie locale (skip curl)
  - Sinon curl + copie dans le cache pour le prochain build

Gain estime : 10-30 minutes par build.
