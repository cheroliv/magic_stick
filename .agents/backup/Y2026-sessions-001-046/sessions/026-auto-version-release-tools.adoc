=== Usage du release workflow

1. GitHub → Actions → "Release Magic Stick" → Run workflow
2. Choisir bump : `patch`, `minor`, ou `major`
3. Le workflow :
   - Bump VERSION
   - Commit + push
   - Cree tag `vX.Y.Z`
4. Le tag declenche `build.yml` → build → verify → test → publish SourceForge


=== Lecons apprises

* Fichier VERSION = pattern standard pour version centralisee
* `workflow_dispatch` avec `type: choice` = UI ergonomique pour bump semver
* Workflow reutilisable (`uses: ./.github/workflows/build.yml`) = pas de duplication
* Un tag GitHub est le declencheur naturel du pipeline de publication
