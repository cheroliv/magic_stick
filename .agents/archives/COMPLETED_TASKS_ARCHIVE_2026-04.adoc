=== Lecons apprises

* bake/ ne doit PAS etre dans .gitignore — c'est dans le layout de build, couvert par build/
* Pas de CNAME pour un site projet GitHub Pages sous domaine personnalise
* Le secret BAKERY_GRADLE_PLUGIN contient le contenu complet du site.yml
* Le pattern d'analyse des repos existants est essentiel pour respecter les conventions
*cname* est non-nullable dans le plugin bakery — ne jamais laisser vide
* Le Gradle wrapper doit etre commit dans le repo pour que la CI fonctionne
* Toujours tester en local (`./gradlew -i publishSite`) avant de deboguer la CI
* Installer les outils dans ~/apps/ en binaire, jamais via apt — fournir un script de mise a jour
* Le token gho_ (fine-grained) fonctionne pour l'API GitHub, le ghp_ (classic) est expire
