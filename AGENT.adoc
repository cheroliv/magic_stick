== Avertissement — Racine de Travail

**Ce fichier est dans `magic-stick/`**, la racine de travail du projet.
Les chemins dans ce fichier sont relatifs a `magic-stick/`. L'agent travaille depuis `magic-stick/`.

== Projet

**Nom** : magic-stick — environnement portable standardise sur cle USB bootable
**But** : Environnement de dev/diagnostic complet, bootable sur tout PC, independant de l'OS hote
**Stack** : Bash, live-build (Debian/Ubuntu ISO customization), xorriso, syslinux/GRUB

=== Cas d'usage

* **Formation** : tous les apprenants ont la meme cle, prets a bosser des le 1er jour
* **Technicien terrain** : diagnostique reseau/FTTH quel que soit l'ordi
* **Dev nomade** : retrouver sa stack complete sur n'importe quel PC
* **Equipe technique** : environnement standardise partage, zero temps perdu en config

=== Architecture des partitions

[cols="1,2,3"]
|===
| Partition | Taille | Role
| System A (ro) | ~8 Go | Systeme live Xubuntu actif (boote)
| System B (ro) | ~8 Go | Systeme live Xubuntu de secours
| Persistence (rw) | Reste de la cle | Donnees utilisateur + config logiciels
|===

**Principe A/B** : Mise a jour sur partition inactive → inversion flag boot → reboot. Rollback si echec.
La partition Persistence n'est JAMAIS touchee par les mises a jour systeme.

== Regles Absolues

_Role : INDEX.adoc = source verite roadmap/portefeuille. AGENT.adoc = regles + conventions. LAZY_EAGER_ESSENTIALS = procedure._

=== 1. Git — INTERDICTION FORMELLE

Commits, push, merge interdits SANS permission explicite.
Autorise (lecture seule) : `git status`, `git diff`, `git log`, `git show`.
Interdit (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`.

=== 2. Commandes destructrices

L'agent NE DOIT JAMAIS lancer `dd`, `mkfs`, `rm -rf` (hors build), `parted`, `fdisk` SANS permission explicite.
Build ISO = 30-60 min : ne PAS lancer automatiquement.

=== 3. Secrets et tokens — INTERDICTION DE LOG

Tokens, mots de passe, credentials NE DOIVENT JAMAIS apparaitre dans le terminal.
* Interdit : passer un token en argument visible (`GH_TOKEN=xxx cmd`), `cat`/`echo` de credentials
* Obligatoire : utiliser stdin/sous-shell pour auth (`(< token gh auth login --with-token)`)
* Lecon Session 008 : token `gho_...` affiche dans terminal pendant stream Discord → compromis en 1s.

=== 4. Fichiers de configuration — REGLE ABSOLUE DE SURETE

Fichiers YAML/JSON/.env non versionnes peuvent contenir des tokens reels.
* Interdit : `Write` complet quand `Edit` partiel suffit, remplacer valeurs sensibles par placeholders
* Obligatoire avant modif : `git check-ignore` + `git ls-files` → si non tracke, CREER `.bak`
* Preserver les valeurs sensibles intactes. Verifier apres.
* Lecon Session 2 (bakery-plugin) : ecraser site.yml non versionne a detruit des tokens reels.

=== 5. Fichiers personnels — Interdiction de modifier

L'agent NE DOIT JAMAIS modifier `~/.gitconfig`, `~/.bashrc`, `~/.ssh/*`, `~/.zshrc`, etc. SANS permission.
L'environnement utilisateur est hors scope projet.
* Lecon Session 033 : l'agent a tente de modifier `~/.gitconfig` → comportement dangereux.

=== 6. Methodologie de Travail

* 1 fichier a la fois + verification immediate (`bash -n`)
* Max 50k tokens EAGER (~3000 lignes)
* 5x20min > 1x2h. Splitter si > 50 messages ou > 3 fichiers modifies en parallele.
* Pas de commentaires dans le code sauf demande.

== Composants Cles

* **live-build** : Construction ISO Debian/Ubuntu customisee
* **Partition A/B** : Mise a jour atomique avec rollback
* **Persistence** : ext4 + `persistence.conf` (/ union)
* **GRUB/syslinux** : Boot A/B selector
* **Convergence versions** : `scripts/lib/versions.sh` (maitre) → hook chroot le source, `Dockerfile` declare ARG alignes
* **Zero sudo** : tout se fait dans le conteneur Docker, jamais sur l'hote

== Anti-Patterns — Pieges a Eviter

|===
| # | Anti-Pattern | Symptome | Prevention
| 1 | **Agent sans `opencode.json`** | Demarre nu, questions basiques | `opencode.json` AVANT `AGENT.adoc`
| 2 | **Trop d'agents trop tot** | Stubs vides, hesite | 100 lignes metier ou 5 regles avant agent specialise
| 3 | **PROMPT_REPRISE fantome** | Mission derivee, pas a jour | Mettre a jour systematiquement en fin de session
| 4 | **Archive vide** | Placeholders "TODO" | Remplir integralement, jamais de TODO
| 5 | **Portefeuille fantome** | Projet non ajoute aux autres INDEX | Ajouter dans TOUS les portefeuilles
| 6 | **Redondance cachee** | Copie-colle AGENT vs INDEX → divergence | INDEX = dashboard pur, AGENT = regles + conventions
| 7 | **Session trop longue** | 2h, 85 messages, erreurs repetees | Splitter > 50 messages ou > 3 fichiers
|===
